## 需求总结
- 增加识别耗时的 UI 展示
- 识别过程中以“聊天消息流”的方式持续展示进度与日志

## 现状调研（已知）
- 前端已收集 debugLogs 并展示为日志面板
- 后端在 /api/analyze 有基础日志输出与 debug 字段

## 实现方案
### 前端
1. 增加识别计时
   - 在开始识别时记录 startTime
   - 结束/失败时计算耗时并展示在顶部状态栏或左侧面板
2. 聊天式进度流 UI
   - 将 debugLogs 改为“消息列表”渲染（角色、时间、内容）
   - 在识别开始、请求发送、后端返回、解析完成等关键节点持续追加消息
3. 可选：自动滚动到最新消息

### 后端
4. 扩展识别过程日志字段
   - 在请求开始/调用模型前/模型返回后/解析完成/异常时生成可返回的 debug 节点
   - 在响应中返回 debugTimeline（供前端渲染为聊天消息）

### 接口与数据
5. 定义统一调试消息结构
   - {time, level, message, data}
   - 前端追加“系统消息”与“模型消息”区分展示

## 验证方式
- 本地上传视频触发识别
- 观察计时是否正确更新
- 观察聊天消息是否按阶段持续出现
- 控制台确认无报错
